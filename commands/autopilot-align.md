# /autopilot-align
与用户深入讨论需求，生成需求对齐文件，确保项目方向正确。

## 使用方法
```bash
/autopilot-align                    # 完整需求对齐流程
/autopilot-align --update          # 更新现有需求对齐
/autopilot-align --verify          # 验证当前需求对齐
```

## 讨论内容
### 原始需求解析
- 核心目标和成功标准
- 功能需求和使用场景
- 技术约束和资源限制

### 技术方案讨论
- 架构设计和技术栈选择
- 实现细节和算法逻辑
- 错误处理和性能考虑

### 质量标准设定
- 代码风格和文档要求
- 测试策略和覆盖率
- 性能指标和安全标准

### 用户偏好确认
- 工作方式详细程度
- 创新程度和风险容忍度
- 沟通方式和决策参与度

## 产出文件
生成项目特定的 `REQUIREMENT_ALIGNMENT.json`：
- 用户原始需求描述
- 确认的技术方案和质量标准
- 用户偏好和约束条件
- 完整的交付物清单

## 验证环节
- 需求理解准确性确认
- 技术方案合理性验证
- 质量标准适当性检查
- 用户最终确认批准

## 使用场景
- 新项目启动时
- 需求发生重大变化时
- 执行方向偏离时
- 定期需求对齐检查

## 相关命令
- `/autopilot-start` - 启动完整工作流
- `/autopilot-status` - 检查对齐状态
- `/autopilot-execute` - 基于对齐执行
